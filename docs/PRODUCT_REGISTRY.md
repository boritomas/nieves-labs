## Status Definitions

### **Operational**
Product is live in production and serving users. Actively maintained with ongoing improvements.

### **Active Development**
Core platform under active development. Foundation features being built for production release.

### **Prototype**
Working prototype or MVP. Experimental phase to validate concept and gather user feedback.

### **Planning**
Product planned but not yet in development. Requirements and design phase.

## How to Use This Registry

1. **Track Product Progress**: Check status column to understand where each product is in its lifecycle.
2. **Identify Dependencies**: See which products depend on the platform before planning launches.
3. **Update Milestones**: Keep "Next Milestone" column current as products progress.
4. **Check Production URLs**: Know which products have live user access.
5. **Understand Ownership**: Identify who is responsible for each product.

---

## Adding New Products

When a new product idea is approved:

1. Add a row to the registry
2. Set Status to "Planning"
3. Set Version to "Concept"
4. Leave Repository and Production URL as "TBD"
5. Define Next Milestone
6. Ensure the product aligns with [PRODUCT_CATALOG.md](./PRODUCT_CATALOG.md) Public Messaging Rules
7. Update Last Updated date
